Do you feel fraustrated on catching exceptions for some methods that are needed? Normally, you always need them to do so.

<Snippet file="snippet-rec-for-kt.mdx" />

<Accordion title="Legacy methods">

<Tabs>

<Tab title="Kotlin">
#### Use `Throws` annotation

```kt
@Throws(Exception::class.java)
fun method() {
    // ...
}
```
</Tab>

<Tab title="Java">
#### Use `throws`

```java
public void method() throws Exception {
    // ...
}
```
</Tab>

</Tabs>

#### Use `try-catch` method
```kt
fun method() {
    try {
        // ...
    } catch (e: Exception) {
        // ...
    }
}
```
</Accordion>

---

## Use `catchException()` function!

It's easy to use, just like a brand new syntactic sugar in Kotlin:

<CodeGroup>
```kt Kotlin
catchException {
    // ...
}
```
```java Java
CommonExtKt.catchException(continuation -> {
    // ...
    return null;
});
```
</CodeGroup>

## Custom exceptions

If you want to implement exceptions, you can do following:

<CodeGroup>
```kt Kotlin
catchException(
    catchAction = { e -> Toast.makeText(this, e.message, Toast.LENGTH_SHORT) }
) {
    // try ...
}
```

```java Java
CommonExtKt.catchException((e, continuation) -> {
    Toast.makeText(this, e.getMessage(), Toast.LENGTH_SHORT);
    return null;
}, continuation -> {
    // try
    return null;
});
```
</CodeGroup>
